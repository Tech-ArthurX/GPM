# GS 语言规范

英文版: [SPEC.en.md](SPEC.en.md)

版本：2026-06-29

这是 GS 的规范性约定，描述解析、执行和宿主交互的基本规则。
命令示例见 `docs/reference.md`，入门说明见 `docs/tutorial.md`。

## 范围

GS 是一门行式自动化语言，给 GPM 包脚本和独立编译工具使用。
它支持包阶段、文件系统、进程执行、注册表访问、网络 I/O、JSON 和 XML、归档操作和编译器扩展。

## 源码规则

- 一行物理文本就是一条逻辑语句，原始代码块除外。
- 推荐使用 UTF-8。
- 预处理器可以容忍 UTF-8 BOM。
- 不提供通用续行语法。

预处理会先归一化这些形式：

- 缩进式 `func` 块会变成 `FUNC ... _END`
- 缩进式 `if / elif / else` 会变成 `WHEN ... _END`
- `CGSB/CGSE`、`PGCB/PGCE`、`AGCB/AGCE` 会被折叠成单个参数

## 词法规则

空白会分隔 token，换行会结束语句。

当 `;`、`#` 或 `//` 出现在字符串外的下一个 token 位置时，它们会开始注释。
`http://...` 和 `https://...` 这种 URL 可以直接裸写。
如果参数必须以 `//` 开头，就应当加引号。

只支持双引号字符串。
支持的转义是 `\n`、`\t`、`\"` 和 `\\`。

裸词会在逗号、等号、空白、换行、回车或引号前结束。
多个裸词在下一个逗号前会合并成一个参数。

## 语句形式

```gs
COMMAND arg1, arg2, key=value
```

- 运行时命令通常使用 4 个大写字母，比如 `SETV`、`FILE`、`HTTP`、`LOGS`。
- 长大写名字保留给宿主指令和显式扩展，比如 `INSTALLROOT`。
- `_END` 用来结束块命令。
- 逗号用于分隔参数。
- `KEY=VALUE` 是命令约定，不是独立的语言级赋值语法。

未知运行时命令应警告后继续。
未知编译器命令应失败。

## 变量与展开

GS 会在执行前展开变量。

```gs
%ENV_NAME%
%@LOCAL_NAME%
```

- `%NAME%` 读取进程环境变量。
- `%@NAME%` 读取 GS 局部变量。
- 普通展开里，没找到的值会变成空字符串。
- `SETV`、`CALC`、`ENVI` 和很多数据类命令会写入局部变量。

## 表达式

`WHEN`、`IFEX` 和缩进式条件都使用布尔表达式。

从低到高的优先级是：

- `OR`
- `AND`
- `NOT`
- `==`、`!=`、`>`、`<`、`>=`、`<=`

支持括号。
当两边都能转成数字时就按数字比较，否则按字符串比较。
空字符串、`0`、`false`、`FALSE`、`no`、`NO` 视为假值。

## 路径

- 空路径表示脚本目录。
- 绝对路径会清理后直接使用。
- 相对路径基于当前 `.gs` 文件解析。
- `EXEC` 的工作目录是脚本目录。

包宿主可以再加额外安全检查，比如阻止 zip-slip。

## 控制流

推荐的子程序写法：

```gs
FUNC setup
  LOGS INFO, preparing

CALL setup
```
